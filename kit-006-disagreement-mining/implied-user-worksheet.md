# Implied-User Worksheet

Contradictions between systems don't resolve in the abstract — they resolve against a specific life. This worksheet converts each surviving fight into a decision.

## Step 1 — Map each author's implied user

From query 2's output, fill one row per loaded thinker:

| Author | Implied user (job type) | Era / environment | Core constraint their system fixes |
|---|---|---|---|
| | | | |
| | | | |
| | | | |

Example rows from our run: Allen → late-90s executive → drowning in incoming logistics → open loops. Forte → 2020s digital creator → fragmented 15-min-to-3-hr windows → content overload. Newport → maker/researcher → distraction-saturated workplace → eroded capacity for unbroken thought.

## Step 2 — Write your own row

| You | Your actual job | Your actual environment | Your binding constraint |
|---|---|---|---|
| | | | |

One honest sentence per cell. "Knowledge worker" is not a job — what does your best day produce?

## Step 3 — Resolve each fight against your row

For each contradiction the collision surfaced:

```
Fight: ______________________________________
Author A's position assumes a user who: ______
Author B's position assumes a user who: ______
I am closer to: ______
So for me the answer is: ______
Exception (the part of my week where the OTHER answer is right): ______
```

The exception line matters. Most people are Allen's user for two hours a day and Newport's for two others — per-project or per-block resolution beats picking a team.

## Step 4 — The sentence

Finish this before adopting anything new from any of them:

> "________'s system was designed for ________, and I am / am not that person, because ________."

If you can't finish it, that's the finding — go back to their canonical piece and read it looking only for who it assumes you are.
