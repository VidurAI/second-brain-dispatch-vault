# The 4D Processing Checklist — touch it once

Attribution: the 4D rule (Do / Delegate / Date / Delete) is paraphrased from a video by the creator whose corpus this publication is built on (a 1.2M-subscriber ex-CEO); the framing below adapts it for second-brain intake. The underlying "touch each item once" discipline also echoes GTD's two-minute rule (David Allen).

## The rule

Every item that enters your capture inbox gets touched **exactly once**, and leaves the inbox through one of four doors:

1. **Do** — it takes under ~10 minutes to extract the value (read it, distill one note, act on it)? Do it now, in the processing window.
2. **Delegate** — someone or something else should process it? Hand it off explicitly (including: "the weekly synthesis job will consume this" — that's delegation to a system, and it only counts if the system actually runs on a schedule).
3. **Date** — genuinely needs future-you? Snooze it to a *specific* time, not to "later." An undated later is a graveyard plot with a reservation.
4. **Delete** — be honest: would you re-save this today? If not, it goes. Deleting is not failure; it's maintenance. Storage is free, but every dead item is camouflage the live ones have to compete with.

## The ritual

- Batch processing into 2–3 fixed windows per week (fixed time + place — see `calendar-template.md` for why the wording matters).
- In the window: newest to oldest, one decision per item, no re-reading the whole thing "to be sure."
- Never leave an item half-open. Unfinished items keep draining attention after you close the tab (the Zeigarnik effect — open loops persist in working memory).
- Track one number per session if you like: items touched vs. items deleted. A healthy intake deletes a lot.

## The tell you're doing it wrong

If your inbox count only ever goes up, you're not processing — you're collecting with extra steps. That's the collector's fallacy wearing a productivity costume (Tietze, 2014).
